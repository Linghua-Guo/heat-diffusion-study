# Results Index

The `results/` directory is organized by purpose:

```text
results/
  baselines/      canonical single-run outputs
  experiments/    mesh/strength/residual studies
  comparisons/    FD vs FEniCSx aligned-grid comparisons
```

Smoke tests and cache outputs have been removed from the canonical results tree.

## Baselines

| Directory | Meaning |
|---|---|
| `baselines/fd_single` | 2D FD square source baseline, `n=64` |
| `baselines/fd_gaussian` | 2D FD single Gaussian source baseline, `n=64` |
| `baselines/fd_two_gaussian` | 2D FD two Gaussian source baseline, `n=64` |
| `baselines/fd_3d_gaussian` | 3D FD Gaussian source baseline, `n=64` |

Each FD baseline may contain local array data such as `solution.npz`, but Git tracks only lightweight reproducible outputs:

```text
summary.csv
temperature.png
```

Binary field data such as `solution.npz`, `temperature.h5`, and `temperature.xdmf` are ignored by git and can be regenerated by rerunning the scripts.

The 3D baseline additionally contains:

```text
temperature_midplane.png
temperature_3d.png
```

## Experiments

| Directory | Meaning |
|---|---|
| `experiments/fd_experiment_single` | square source mesh/strength/residual study |
| `experiments/fd_experiment_gaussian` | Gaussian source mesh/strength/residual study |

Each experiment directory contains:

```text
mesh_study.csv
mesh_study.png
strength_study.csv
strength_study.png
residual_study.png
conclusions.md
```

## FD/FEniCSx Comparisons

| Directory | Meaning |
|---|---|
| `comparisons/fd_compare_n16` | FD square source, `n=16` |
| `comparisons/fd_compare_n32` | FD square source, `n=32` |
| `comparisons/fd_compare_n64` | FD square source, `n=64` |
| `comparisons/fenicsx_compare_n17` | FEniCSx square source summary, mesh subdivisions `n=17` |
| `comparisons/fenicsx_compare_n33` | FEniCSx square source summary, mesh subdivisions `n=33` |
| `comparisons/fenicsx_compare_n65` | FEniCSx square source summary, mesh subdivisions `n=65` |
| `comparisons/fd_fenicsx_comparison` | Combined comparison table |

The comparison aligns grid scale by using FEniCSx `n = FD n + 1`, since FD uses interior unknowns with $h=1/(n+1)$ while FEniCSx uses mesh subdivisions with $h=1/n$.

Core comparison file:

```text
comparisons/fd_fenicsx_comparison/comparison.csv
```
